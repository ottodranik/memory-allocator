Алгоритм работы с хэдером:
 * Пример:
 * Инициализируется хэдер со значением: 25215009 = 00000001100000001100000000100001
 * Где:
 * + последние 3 бита указывают на свободное место в последнем DWORD слове блока.
 *   единицы измерения: от 000 до 111 -> т.е. от 0 до 7 свободных байтов (в зависимости от значения DWORD)
 *   в примере: 1 = 1 свободных байт
 * + следующие 10 бит указывают на размер блока в DWORDах
 *   единицы измерения: DWORD
 *   в примере: 0000000100 = 4
 * + следующие 10 бит указывают на сдвиг от начала дампа или "начальный локальный адрес блока" (с какого по счёту слова (DWORD) от начала дампа начинается блок)
 *   единицы измерения: DWORD
 *   в примере: 0000000110 = 6 - это значит, что ДО этого блока места с 1 по 5 заняты DWORDами
 * + следующий 1 бит является флагом и указывает свободен блок или занят
 *   в примере: 1 = 1 - занят
 * + следующий 1 бит является флагом и указывает на сам хэдер. Необходим для поиска блоков в дампе
 *   в примере: всегда равен 1

Далее, хэлперы для работы с хэдером блока:

        int get_block_free_size(int num)
        int get_block_size(int num)
        int get_block_address(int num)
        int get_block_busy_state(int num)
        int get_block_head_byte(int num)

делают побитный сдвиг и сравнение для нахождения всех описанных выше значений.


NOTE! Почему тип функций - указатель на void?
 
        void *mem_alloc(size_t size)
        void *mem_realloc(void *addr, size_t size)
        void mem_free(void *addr)

 * https://stackoverflow.com/a/16583761
 * A pointer to void can point to anything, it's a generic nondescript
 * anonymous pointer to some memory. This means that you can pass any kind 
 * of pointer as the first argument of the function, as all pointers can implicitly 
 * be converted to void*.
